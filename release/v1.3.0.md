# v1.3.0

Release automation update for `unit-converter`.

## Highlights

- Added a GitHub Actions release workflow that builds the wheel and source
  distribution, checks them, and uploads them to the matching GitHub Release
  when a `v*` tag is pushed.
- Added `build` and `twine` to the development dependency group so release
  packaging checks can run from a standard project install.
- Documented the tag-based GitHub Release asset publishing flow in
  `release/README.md`.
- Stabilized full-list catalog normalization so generated release checks do not
  reorder existing unit reference data.

## Notes

- Runtime conversion behavior is unchanged from v1.2.0.
- The package version is now `1.3.0`; tag the release as `v1.3.0`.
