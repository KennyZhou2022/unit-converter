# v1.2.0

UI catalog and development data release for `unit-converter`.

## Highlights

- Added `get_ui_unit_catalog()` as a public helper for loading the packaged UI
  category tree.
- Added `src/unit_converter/data/ui_unit_catalog.json` as lightweight runtime
  UI data. It stores only display categories and subcategories.
- Added direct unit metadata to `src/unit_converter/data/unit_catalog.json`.
  Every supported unit now has a record in `units` with both `nist_categories`
  and `ui_categories`.
- Replaced the former frequently-used converter grouping with full-list
  top-level categories: Dimension, Mechanics, Heat, Fluids, Light,
  Electricity, Magnetism, Radiology, and Miscellaneous.
- Mapped all 467 currently supported units into the full-list UI taxonomy.
  284 mappings come from direct full-list unit label matches, and 183 use the
  unit's NIST context as the fallback placement.
- Added neutral development reference data under `data/external/`:
  `full_list_categories.json` and `full_list_unit_catalog.json`.
- Added generation scripts for the UI category tree, the development full-list
  unit catalog, and unit-level UI metadata.
- Updated `schemas/unit_catalog.schema.json` for the new `units` metadata
  shape.
- Updated README, API reference, quickstart, and package-data tests for the new
  UI catalog behavior.
- Removed the generated summary table from the Supported Units documentation
  page.
- Updated sdist packaging to include `data/` and `release/` so development
  reference data and release notes are available from source distributions.

## Notes

- Runtime conversion behavior is unchanged from v1.1.0.
- The full-list unit catalog is not used by `convert()`.
- The former frequently-used/common converter grouping is intentionally not
  represented as a top-level UI category in this release. It can be added later
  as a separate featured-unit list without changing the full-list taxonomy.
- `get_ui_unit_catalog()` gives the category tree. Use
  `get_unit_catalog()["units"]` when the UI needs unit labels with their direct
  NIST and UI metadata.
